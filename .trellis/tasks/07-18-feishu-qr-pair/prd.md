# PRD：飞书二维码配对（二期）

## 背景

已有短码配对（`/lark-pair` + 自动引导）。用户希望本机弹出**二维码**辅助配对。一期仅短码；本任务在不改变绑定协议的前提下增加码图展示。

## 目标

- 收到 `pair_challenge` 时：除短码文案外，生成本地 PNG 二维码并尽量用系统打开。
- 二维码载荷为辅助文案：`配对 <CODE>`（与飞书口令一致）；绑定仍靠飞书/control 发送口令。
- 自动引导与手动 `/lark-pair` 共用同一展示路径。
- 生成/打开失败时**降级**为纯短码 notify，不阻断配对。

## 非目标

- 飞书深链预填、公网/局域网 HTTP 扫码直绑
- 云端中继、多主人、去掉短码

## 已决

| 决策 | 选择 |
|------|------|
| 扫码语义 | **A：辅助展示**（载荷 `配对 <码>`） |
| 展示载体 | **A：`~/.pi/lark-hub/pair-qr.png`** + notify 路径 + 尝试系统打开 |
| 依赖 | **A：允许** 在 dependencies 增加轻量库（如 `qrcode`）以写 PNG |

## 验收

1. `/lark-pair` 或自动引导出码后：存在可扫 PNG（固定路径可覆盖），notify 含码、口令与路径。
2. 系统打开尽力而为；失败不抛错、不阻断。
3. 绑定协议不变：飞书/control「配对 CODE」+ openId → 单主人落盘。
4. QR 失败仅短码文案仍可用。
5. `npm run typecheck` + 相关单测通过。
